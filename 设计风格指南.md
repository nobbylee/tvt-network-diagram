# 设计风格指南

> 面向开发者：新增页面/组件时的视觉规范依据，保证整套系统"看起来是一个人做的"。
> 源文件：`client/src/styles/tokens.css`（变量+全局样式）、`client/src/styles/notebook.css`（前台通用组件类）、`client/src/styles/element-overrides.css`（管理后台 Element Plus 主题化）。改视觉先改这三个文件，不要在业务组件里写死颜色/字号数值。

---

## 一、设计语言：复古知识清单（Notebook Infographic）

整体基调是"教材编辑排版 + 手账笔记本"：暖色纸面、左侧装订线、横向内页格线、衬线标题、等宽数据、勾选清单式的卡片交互。刻意避开常见后台系统的冷色调科技感，也避开陶土橙这类过度装饰的"文创风"，走**克制的教材感**。

两端共用同一套设计 token（颜色/字体/间距完全一致），差异只在组件实现方式：

| 端 | 实现方式 |
|---|---|
| 员工前台 | 纯自定义 HTML + `notebook.css` 里的 `.nb-*` 组件类，不用 Element Plus |
| 管理后台 | Element Plus 组件（表格/表单/弹窗等），但通过 `element-overrides.css` 把配色、圆角、字体全部换成同一套 token，视觉上跟前台统一 |

---

## 二、色彩系统

定义在 `tokens.css` 的 `:root` 里，全部通过 CSS 变量引用，禁止在组件里写死十六进制颜色。

### 纸面 / 背景

| 变量 | 值 | 用途 |
|---|---|---|
| `--bg-app` | `#EFEBE3` | 页面最外层背景（略冷的教材纸色，不是暖橙） |
| `--bg-surface` | `#FBF8F1` | 卡片/表格/输入框等"纸张"表面 |
| `--bg-sunken` | `#E7E1D4` | 凹陷区域，如表头背景、标签底色 |
| `--bg-hover` | `#F3EEE4` | 悬浮态背景 |
| `--bg-rule` | `rgba(42,58,78,0.08)` | 页面横向内页格线（很淡，营造"练习本"质感） |
| `--bg-margin` | `#F6EDEA` | 装订线/边距相关的浅底色 |

### 墨水与强调色

| 变量 | 值 | 用途 |
|---|---|---|
| `--ink` | `#1C2838` | 最深的正文墨色，等同 `--text-primary` |
| `--ink-soft` | `#3D4A5C` | 次深墨色，用于表头、次级标题 |
| `--accent` | `#2F5D50` | **品牌主色**，清单勾选绿；按钮、链接、选中态都用它 |
| `--accent-hover` | `#244A40` | 主色悬浮/激活态 |
| `--accent-soft` | `#E2EDE8` | 主色浅底（选中背景、hover 背景） |
| `--margin-rule` | `#C45C4A` | 笔记本左侧装订红线，`.nb-sheet--margin` 用它做左边框 |
| `--highlight` | `#F3E7B4` | 荧光笔浅黄，用于 `.nb-kicker` 小圆点等强调点缀，**慎用，不做大面积底色** |

### 文字

| 变量 | 值 | 用途 |
|---|---|---|
| `--text-primary` | = `--ink` | 正文/标题主文字 |
| `--text-secondary` | `#5A6574` | 次要说明文字 |
| `--text-muted` | `#8A93A0` | 弱化文字（时间戳、占位符、禁用态） |
| `--text-on-accent` | `--bg-surface` | 深色/主色按钮上的文字 |

### 边框

| 变量 | 值 | 用途 |
|---|---|---|
| `--border` | `#D5CFC2` | 默认描边 |
| `--border-strong` | `#B8B0A0` | 强调描边（hover、选中） |

### 语义色（状态）

| 变量 | 值 | 用途 |
|---|---|---|
| `--success` / `--success-soft` | `#2F5D50` / `#E2EDE8` | 成功、空闲、通过（与主色共用，因为主色本身就是"勾选绿"语义） |
| `--danger` / `--danger-soft` | `#A3483C` / `#F6EDEA` | 危险操作（删除、解除占用、强制释放），与 `--margin-rule` 同色系 |
| `--warning` / `--warning-soft` | `#9A6B2F` / `#F6EBD4` | 警告、占用中提示 |
| `--info` | `#3F5F7A` | 中性提示信息 |

**用色原则**：正文永远用墨色系（`--ink`/`--text-*`），不要用纯黑 `#000`；强调/交互用 `--accent`，不要新引入别的彩色；状态色只用于真正表达状态（成功/危险/警告），不要拿来做装饰。

## 三、字体与排版

三套字体各司其职，混用是刻意设计，不要图省事全用一种：

| 变量 | 字体栈 | 用在哪 |
|---|---|---|
| `--font-serif` | Source Serif 4 / Noto Serif SC / 宋体 / Georgia | **标题**（`h1/h2/h3`、`.nb-title`）——教材感的来源 |
| `--font-sans` | IBM Plex Sans / 苹方 / 微软雅黑 | **正文**、按钮、表单，`body` 默认字体 |
| `--font-mono` | IBM Plex Mono / SF Mono / Consolas | **数据类文本**：时间戳、文件大小、账号、序号、标签（`.nb-kicker`、`.mono-text`） |

### 字号阶梯

| 变量 | 值 | 典型场景 |
|---|---|---|
| `--fs-display` | 40px | 首页大标题级别，极少用 |
| `--fs-h1` | 28px | **页面主标题**，管理后台所有页面标题统一用这个（见下方"踩过的坑"） |
| `--fs-h2` | 20px | 卡片标题、二级标题 |
| `--fs-h3` | 16px | 列表项标题（如远程桌面卡片里的设备名） |
| `--fs-body` | 15px | 正文默认字号 |
| `--fs-sm` | 13px | 次要文字、mono 数据文本 |
| `--fs-xs` | 12px | 标签、meta 信息、徽标文字 |

> **注意**：字号变量只有以上 7 个，不存在 `--fs-xl`、`--fs-lg` 这类变量。此前有页面误写了不存在的 `--fs-xl`，CSS 无法解析导致标题退化成继承的小字号——加新样式前请对照这张表，变量名拼错不会报错，只会静默失效。

### 行高与字重

| 变量 | 值 | 用途 |
|---|---|---|
| `--lh-tight` | 1.25 | 标题行高 |
| `--lh-normal` | 1.65 | 正文行高，`body` 默认值 |
| `--fw-normal` / `--fw-medium` / `--fw-semibold` | 400 / 500 / 600 | 不用 700+ 的粗体，教材感的克制字重 |

## 四、间距与圆角

### 间距（8px 网格，`--sp-1` 到 `--sp-8`）

```
--sp-1: 4px   --sp-2: 8px   --sp-3: 12px  --sp-4: 16px
--sp-5: 24px  --sp-6: 32px  --sp-7: 48px  --sp-8: 64px
```

组件内边距、元素间距一律用这套变量，不写魔数（比如不要写 `padding: 18px`，往上取 `--sp-4`(16px) 或 `--sp-5`(24px)）。

### 圆角

| 变量 | 值 | 用途 |
|---|---|---|
| `--radius-sm` | 2px | 按钮、输入框、标签——**系统里绝大多数圆角用这个**，克制、不圆润 |
| `--radius-md` | 4px | 卡片、表格容器（`.nb-sheet`） |
| `--radius-lg` | 6px | 极少用，大容器 |
| `--radius-full` | 9999px | 胶囊/圆形，仅特殊场景 |

**这套系统的圆角整体偏小**——不是常见后台系统那种大圆角卡片感，要的就是"纸张直角"的克制。新组件默认用 `--radius-sm`，除非是卡片容器才用 `--radius-md`。

## 五、阴影与动效

| 变量 | 值 | 用途 |
|---|---|---|
| `--shadow-card` | `0 1px 0 rgba(28,40,56,.04), 0 8px 20px rgba(28,40,56,.05)` | 卡片默认阴影，很浅，像纸张微微悬浮 |
| `--shadow-pop` | `0 8px 24px rgba(28,40,56,.08)` | 弹出层（下拉、弹窗）阴影，比卡片阴影明显一点 |
| `--shadow-none` | `none` | 显式去阴影，用于覆盖 Element Plus 默认阴影（如 `.el-card`） |

动效统一用：

```css
--ease: cubic-bezier(0.2, 0, 0, 1);
--dur-fast: 140ms;   /* 悬浮态等即时反馈 */
--dur-base: 220ms;   /* 弹窗、下拉等过渡 */
```

不用弹簧类缓动（bounce/elastic），保持教材式的稳重克制。

## 六、页面底纹：笔记本纸感

`body` 的背景不是纯色，而是叠了两层渐变，模拟练习本内页：

```css
background-image:
  /* 左侧装订红线，距左边 47px */
  linear-gradient(90deg, transparent 47px, var(--margin-rule) 47px, var(--margin-rule) 49px, transparent 49px),
  /* 每 32px 一条极淡横线 */
  repeating-linear-gradient(to bottom, transparent 0, transparent 31px, var(--bg-rule) 31px, var(--bg-rule) 32px);
background-attachment: fixed;
```

这是全局唯一一处，不要在业务页面里重复实现，也不要因为某个页面"看起来空"就自己加背景纹理。

## 七、通用组件类（`notebook.css`）

前台页面搭界面首选这些现成类，而不是每个页面重新写卡片样式：

| 类名 | 效果 |
|---|---|
| `.nb-sheet` | 基础卡片：纸面背景 + 描边 + 圆角 + 卡片阴影 |
| `.nb-sheet--margin` | 叠加左侧 3px 装订红线（`--margin-rule`），列表行、内容区块常用 |
| `.nb-sheet--ruled` | 叠加横向内页格线背景 |
| `.nb-title` | 衬线字体标题，配合 `h1/h2` 使用 |
| `.nb-subtitle` | 次级说明文字，`--text-secondary` |
| `.nb-kicker` | 小型分类标签，等宽字体 + 字间距 + 一个方形勾选装饰点 |
| `.nb-check-card` | **可点击卡片**：左侧红线条 + hover 上浮 2px + 边框变深，首页功能入口卡片用的就是这个 |
| `.nb-check` | 配合 `.nb-check-card` 的方形勾选框图标，hover 时变主色 |
| `.nb-meta-row` | 一行两端对齐的 meta 信息条（如"序号 ⟷ 时间"），等宽字体 |
| `.nb-tag` | 荧光笔黄底小标签 |
| `.nb-divider` | 虚线分隔线（不是实线，用短横线重复模拟手绘感） |

**新页面起步模板**（参照 `client/src/views/student/RemoteDesktops.vue` / `CommonTools.vue`）：

```vue
<section class="page-hero nb-sheet nb-sheet--margin">
  <router-link to="/" class="back-link">← 返回首页</router-link>
  <h1 class="page-title nb-title">页面标题</h1>
</section>

<div v-for="item in items" :key="item.id" class="row nb-sheet">
  <div class="nb-meta-row">...</div>
  ...
</div>
```

## 八、管理后台：Element Plus 主题化

后台用 Element Plus 组件库，但**不用它的默认主题**，`element-overrides.css` 把关键 CSS 变量全部映射到上面这套 token：

- `--el-color-primary` → `--accent`（绿）
- `--el-border-radius-base` → `--radius-md`，`--el-border-radius-small` → `--radius-sm`
- `--el-table-header-bg-color` → `--bg-sunken`，表头字体强制换成衬线 `--font-serif` + `--fw-semibold`
- `.el-card` 去掉默认阴影，换成描边 + 卡片阴影
- 输入框统一用 `box-shadow: 0 0 0 1px var(--border) inset` 模拟描边，而不是 Element 默认的 border

**后台页面标准结构**（参照 `client/src/views/admin/RemoteDesktopList.vue` / `CommonToolList.vue`）：

```vue
<div class="xxx-admin-page">
  <div class="page-header">
    <h1 class="page-title nb-title">页面标题（两个字+"管理"，如"工具管理"）</h1>
    <el-button type="primary">主操作</el-button>
  </div>

  <div class="table-panel nb-sheet nb-sheet--margin">
    <el-table ...>...</el-table>
  </div>
</div>
```

```css
.page-title {
  margin: 0;
  font-size: var(--fs-h1);   /* 全站统一用这个，不要用 --fs-xl（不存在）*/
}
```

**页面标题命名规范**：后台侧边栏和页面 `<h1>` 统一用「两字业务名 + 管理」（题库管理/成绩管理/考试管理/资料管理/日报管理/远程管理/工具管理/账号管理），不要用「远程桌面管理」这种四字+管理的长名字，也不要侧边栏和页面标题文案对不上。

## 九、组件写作规范（Dos & Don'ts）

- ✅ 颜色/字号/间距/圆角全部引用变量：`color: var(--text-secondary)`
- ❌ 不写死数值：`color: #5A6574`、`font-size: 13px`
- ✅ 数据类文本（时间、ID、文件大小、账号）用 `font-family: var(--font-mono)`，视觉上跟正文区分开
- ✅ 表格/列表单元格如果内容是定长格式（如 `2026-07-11 18:00:00`），给列足够宽度或加 `white-space: nowrap`，避免换行挤压行高
- ✅ 危险操作（删除、解除、强制释放）配 `--danger` 系配色 + 二次确认弹窗（`ElMessageBox.confirm`）
- ❌ 不引入这套 token 之外的新颜色、不用大圆角、不用重阴影、不用高饱和撞色
- ✅ 新建管理后台表格页面时，各列宽度加总不要超过容器实际可用宽度（内容区约 1000-1200px，减去 padding），否则会出现横向溢出/换行拥挤，宁可列窄一点也不要让表格溢出
