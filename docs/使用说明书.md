# TVT 网络架构图绘制工具 — 使用说明书

> 适用版本：v1.0  
> 更新日期：2026-07-09  
> 适用对象：技术支持、研发、售前工程师

---

## 1. 产品简介

本工具用于快速绘制、保存、分享客户现场的安防网络拓扑图，内置 TVT / 通用网络设备元件库，操作接近思维导图：拖拽设备、连线、填写 IP 与端口即可。

**典型场景：**
- 客户现场出问题，需要在本地还原网络连接关系
- 团队内共享拓扑，避免口头描述不一致
- 出差时导出 `.narch` 文件离线编辑，回办公室再上传

## 2. 快速开始

### 2.1 开发环境（本地）

**环境要求：** Node.js 20+（推荐 22）

```bash
# 1. 启动后端
cd packages/server
npm install
npm run dev
# → http://localhost:3001

# 2. 另开终端，启动前端
cd packages/web
npm install
npm run dev
# → http://localhost:5173
```

浏览器打开 http://localhost:5173 ，使用下方默认账号登录。

### 2.2 Docker 一键部署（内网服务器推荐）

**环境要求：** Docker + Docker Compose

```bash
# 在项目根目录
docker compose up -d --build
```

浏览器打开 http://服务器IP:8080

数据持久化在 Docker volume `tvt_data` 中。修改密钥：

```bash
JWT_SECRET=请换成足够长的随机字符串 docker compose up -d --build
```

停止服务：

```bash
docker compose down
```

## 4. 功能使用指南

### 4.1 登录

1. 打开系统首页，进入登录页  
2. 输入用户名、密码  
3. 登录成功后进入「我的项目」列表  

右上角可退出登录（若界面有退出入口；清除浏览器本地存储亦可退出）。

### 4.2 项目列表

| 操作 | 说明 |
|------|------|
| **搜索** | 按项目名 / 客户名筛选 |
| **新建项目** | 创建空白拓扑并进入编辑器 |
| **打开项目** | 点击卡片进入编辑器 |
| **删除** | 删除前会二次确认，不可恢复 |
| **导入 .narch** | 从本地文件导入为新项目 |

### 4.3 绘制拓扑

#### 添加设备
1. 左侧「元件库」按分类展开（TVT / 通用网络）  
2. 将设备拖到中间画布，松开即可放置  
3. 拖拽过程中画布会出现蓝色虚线提示区  

#### 连线
1. 将鼠标移到设备节点右侧圆点  
2. 按住拖到另一设备左侧圆点，松开完成连线  
3. 点击连线，在右侧属性面板填写端口、连接类型（网线 / 光纤 / PoE / WiFi / 4G）  

#### 编辑属性
- **设备：** 名称、品牌、型号（选填）、IP（选填）、备注（选填）  
- **连线：** 源端口、目标端口、连接类型、备注  
- 属性修改即时反映到画布；空字段不在节点上占位  

#### 画布操作

| 操作 | 方式 |
|------|------|
| 平移画布 | 鼠标中键 / 右键拖拽，或触控板滚动 |
| 缩放 | 工具栏 +/-，或画布左下角控件 |
| 水平对齐 | 拖动设备接近其他设备同一高度时自动吸附，并显示蓝色辅助线 |
| 框选 | 左键拖拽空白区域 |
| 水平对齐吸附 | 拖动设备接近其他设备同一高度时自动吸附，并显示蓝色辅助线 |
| 删除 | 选中节点/连线后按 `Delete` / `Backspace` |
| 撤销 | `Ctrl+Z`（Mac：`⌘+Z`） |
| 重做 | `Ctrl+Y` 或 `Ctrl+Shift+Z` |
| 折叠侧栏 | 工具栏「元件库 / 属性」按钮，扩大画布 |

### 4.4 自定义元件库

1. 元件库标题旁点击 **+**，或分类旁的 **+**  
2. 填写名称、选择分类与图标形态，保存  
3. 悬停元件可 **编辑 / 删除**  
4. 自定义数据保存在浏览器本地；删除元件库项 **不会** 删除画布上已放置的设备  

### 4.5 保存与分享

| 操作 | 说明 |
|------|------|
| **保存** | 将当前拓扑写入服务器，团队成员可打开 |
| **导出** | 下载 `.narch` 文件，可拷贝到 U 盘出差使用 |
| **导入** | 在列表页导入为新项目；或在编辑器内导入覆盖当前画布（需再点保存同步服务器） |

**.narch 文件** 是 JSON 文本，可用记事本打开，包含项目元信息、设备节点与连线。

## 5. 推荐工作流

### 办公室协作
1. 登录 → 新建或打开项目  
2. 绘制拓扑并填写关键 IP / 端口  
3. 点击「保存」  
4. 同事在项目列表搜索客户名即可打开  

### 出差离线
1. 出发前「导出」`.narch` 到笔记本  
2. 现场可在有服务的环境导入编辑；或带回办公室后「导入」并保存  
3. 注意：导入默认新建项目，避免误覆盖他人正在编辑的版本

## 7. 常见问题

**Q：登录提示无法连接服务器？**  
A：确认后端已启动（开发：`packages/server` 的 `npm run dev`；Docker：容器是否在跑）。

**Q：保存失败 / 401？**  
A：登录可能过期，重新登录后再试。

**Q：导出的文件打不开？**  
A：`.narch` 用本工具「导入」即可；也可用文本编辑器查看 JSON。

**Q：自定义元件刷新后没了？**  
A：自定义元件存在当前浏览器 localStorage。换电脑或清缓存会丢失；画布项目本身保存在服务器，不受影响。

**Q：如何备份全部数据？**  
A：Docker 部署时备份 volume `tvt_data`；或定期导出重要项目的 `.narch` 文件。
