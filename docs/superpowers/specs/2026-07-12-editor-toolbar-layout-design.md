## 1. 背景与问题

`docs/UI设计规范.md`（v1.2，2026-07-09 定稿）设计工具栏时只规划了 6 项：保存 / 导入 / 导出 / 撤销 / 重做 / 缩放 / 项目名称。

此后 v1.5（配置校验）、v1.6（PNG/PDF 导出、文本/箭头标注工具）陆续在 `packages/web/src/components/Toolbar.tsx` 里见缝插针地加入新功能，但设计规范从未同步更新。目前实测 `Toolbar.tsx` 已长到 **520 行、单行容纳 8 组约 20 个控件**（文件操作、编辑操作、缩放、画布工具、实时校验开关+徽标+立即校验、面板折叠），且大部分是纯图标 + hover 提示，无文字标签。

反观 `DeviceLibrary.tsx`、`PropertyPanel.tsx` 的分组、间距、配色仍与规范基本一致，没有明显问题。**结论：界面观感问题不是全局性的，是局部的"功能生长 vs 设计规范未同步更新"，集中在工具栏一处。**

## 2. 目标

- 把顶部工具栏按"文件级操作 / 画布级操作 / 校验状态"三种不同职责拆开，各自归位
- 常用操作恢复文字标签，减少纯图标猜测成本
- 为后续功能（P1-4 批量对齐、P1-5 项目模板等）预留横向空间，不再无限堆砌进一行
- 同步更新 `docs/UI设计规范.md`，避免"功能生长、规范不跟"再次发生

## 3. 非目标（Out of scope）

- 不改变任何 Zustand store（`canvasStore` / `uiStore` / `validationStore`）的状态结构或方法签名
- 不改变现有交互行为（快捷键 Ctrl+S / Ctrl+Z / Ctrl+Y、点击外部关闭导出菜单等）
- 不涉及元件库、属性面板、设备节点、连线样式的视觉调整（现状良好，维持不变）
- 不在本设计范围内处理代码结构拆分（属于子项目 B）和文档整合（属于子项目 C）

## 4. 设计方案：三层布局

```
┌──────────────────────────────────────────────────────────┐
│ 顶栏 44px  返回 | 项目名·客户   保存 导入 导出▾        面板折叠×2│
├──────────┬───────────────────────────────────┬───────────┤
│          │  ┌─────────────────────────┐      │           │
│ 元件库    │  │      画布区域            │      │  属性面板  │
│ 224px    │  │  (浮动工具条悬浮底部居中)  │      │  256px    │
│          │  │ ↶↷|⧉📋|▸T→|-100%+       │      │           │
│          │  └─────────────────────────┘      │           │
│          ├───────────────────────────────────┤           │
│          │ 校验状态栏 28px(常驻,点击展开问题面板)│           │
└──────────┴───────────────────────────────────┴───────────┘
```

参照 Figma / Miro / draw.io 编辑器的通用分层模式，也呼应 `UI设计规范.md` §1.2 中"画布优先"的既有原则。

### 4.1 顶栏（TopBar）—— 文件级操作

高度 44px（原 48px，随内容精简略微收窄）。从左到右：

| 区块 | 内容 |
|------|------|
| 返回 | 返回项目列表图标按钮（含未保存确认逻辑，行为不变） |
| 面包屑 | "TVT 网络架构图 / 项目名" |
| 分隔线 | |
| 保存 | 图标 + **文字**"保存"（原已有文字，保留） |
| 导入 | 图标 + **文字**"导入" |
| 导出▾ | 图标 + **文字**"导出"，下拉菜单三项不变：导出 .narch / 导出 PNG / 导出 PDF |
| 状态提示 | 临时文案（"已保存"/错误提示），行为不变 |
| （弹性空白） | `flex-1` |
| 面板折叠 | 元件库折叠、属性面板折叠两个图标按钮，靠右对齐 |

**从顶栏移出**：撤销/重做、复制/粘贴、缩放、画布工具（选择/文本/箭头）、校验开关+徽标+立即校验。

### 4.2 画布浮动工具条（CanvasToolbar，新增）

新组件，`absolute` 定位于画布容器内，底部居中悬浮，圆角胶囊 + 阴影（视觉参考 Figma 底部工具条）。从左到右，用分隔线分四组：

1. 撤销 / 重做
2. 复制 / 粘贴
3. 画布工具：选择（V） / 文本框（T） / 箭头标注（A），当前 active 态高亮不变
4. 缩放：缩小 / 百分比（点击重置） / 放大

这四组操作的共同点是"作用于画布内容本身"，与顶栏的"文件级"操作分属不同层级，因此单独成组。所有底层事件（`window.dispatchEvent(new Event('tvt-zoom-in'))` 等）和 store 调用保持不变，只是 JSX 挪家。

### 4.3 校验状态栏（复用现有 `ValidationPanel`，非新建）

`Editor.tsx` 中 `ValidationPanel` 已经渲染在画布下方（见 `packages/web/src/pages/Editor.tsx:121`）。本设计不新增组件，而是把 `Toolbar.tsx` 里的校验开关、错误/警告徽标、"立即校验"按钮，**移到 `ValidationPanel` 的常驻头部**：

| 内容 | 行为 |
|------|------|
| 实时校验开关 | 同现有 `validationEnabled` / `setValidationEnabled` |
| 错误徽标（红）/ 警告徽标（橙） | 同现有计数逻辑，>0 时显示 |
| "立即校验"按钮 | 同现有 `bumpRun()` |
| 展开/收起箭头 | 复用现有 `panelOpen` 状态，点击展开下方问题详情列表 |

头部高度 28px，始终可见（不随面板展开/收起消失），作为校验状态的常驻入口。

## 5. 组件与数据流

| 组件 | 变化 |
|------|------|
| `Toolbar.tsx` → 拆分/重命名 | 瘦身为 `TopBar.tsx`，只保留 4.1 的内容和对应的 state（`status`/`saving`/`exporting`/`exportMenuOpen` 等文件操作相关状态） |
| `CanvasToolbar.tsx`（新） | 承接撤销/重做/复制粘贴/画布工具/缩放的 JSX 和对应 store 订阅，渲染在 `CanvasArea.tsx` 容器内 |
| `ValidationPanel.tsx` | 头部新增 4.3 描述的常驻控件区，内部详情列表逻辑不变 |
| `canvasStore.ts` / `uiStore.ts` / `validationStore.ts` | **不变**，所有状态和方法签名保持原样，纯搬家不改数据结构 |

因为不改数据流，这是一次纯展示层重排，风险低。
